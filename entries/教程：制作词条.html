方法一不适合手机、平板等用户，仅推荐使用电脑。

其实方法二也不太适合，尤其是手机。

缺少文件。缺少外观样式表和图标文件，没了这些网页就只是一个天蓝色背景和普普通通的文字，也会失去排版布局和设计好的外观。

开始

有需要可以下载。

↓空白模板文件包の压缩包下载↓

新建一个文本文档，暂时不用管名字。

进去之后在里面复制粘贴（最好是复制粘贴）或者手打输入下面这些内容：

<!DOCTYPE html>
<html>
    <head>
        <meta charset="UTF-8" />
        <title>title</title>
        <link rel="icon" href="../assets/site-icon.ico" />
        <link rel="stylesheet" href="../config/outlook.css" />
    </head>
    <body bgcolor="#6190FF">
        <div id="container">
            <div id="head">
                <h1 id="pagetitle">😅流汗黄豆百科</h1>
            </div>
            <div id="main">
                <div id="sidebox">
                    <a class="sblink" href="">刷新</a>
                    <a class="sblink" href="#">回到顶部</a>
                </div>
                <div id="body">
                    <h1 id="contenttitle"></h1>
                    content
                </div>
            </div>
        </div>
    </body>
</html>

上面标了蓝色的是要替换成你自己的东西：

  • title  →  你的词条的标题，也就是你的词条的名称
  • content  →  页面内容，待会儿马上讲

然后，保存文件都会吧？以Windows的记事本为例，将文件保存在任意位置，到时候找得着就行。

注意保存的时候名称就用你的词条名后面加个.html，文件名下面的“保存类型”点一下，弹出来的列表里选择“所有类型（*.*）”。

注意事项

词条名的命名有一定限制。

  • 名称不能超过30个字符（包括所有类型的符号，不过像&nbsp;这种转义字符实体在代码中占多个字符但是显示出来只占一个字符，算一个字符。
  • 名称如果包含连续的空格，则需要把空格（每一个空格）替换成转义字符实体&nbsp;。此外在名称中最好还是要替换的有：
    • # → &#35;
    • / → &#47;
    • . → &#46;
  • 不要使用像“这样做香翅捞饭，鸡肉咸香又带有一丝甘甜，太下饭了！”这样的像短视频和博文小文章的诡异标题。你想想百科词条该用什么名字。
  • 名称里不能包括换行。你现在也应该不会在HTML里换行才对。
  • 教程类词条标题格式应该是“教程：（你的教程名）”，就像这个教程一样。
  • 避开使用emoji表情、怪异或者少见不常用的符号、Tab、链接、有误导性或法律性的符号 、网址等。
  • 内容不合适也会打回。

编写内容

注：每行代码前面请添加20个空格保持对齐。代码中的空格也需要添加。

所有部件（元素）的代码其实除了部件内容外代码形式都是挺相似的。下面介绍一下部件代码都是如何组成的。

普通形式

<元素名 属性名="属性值">内容</元素名>

无内容纯标签形式

<元素名 />

嵌套形式

<元素名 属性名="属性值">
    内容
</元素名>

上面的“属性名”和“属性值”是成对的，以下简称属性。

一般来说一个元素可以没有属性，不一定要写属性；一个元素也可以有多个属性，不同属性之间用空格隔开来；但是某些特殊的就一定得有属性，一定要有属性的下面会讲到。

属性有很多种，每一种的功能也当然是不一样的，下面介绍的元素也都会写可以用哪些属性或必须用哪些属性，以及使用的场合。

元素（部件）代码列表

标红色的是必要属性、紫色的是可选属性、蓝色的是内容。属性值里的“value”则是可以替换值的地方。

文本类

  • 标题文本

    用于标题的文本，亦可用于在目录中定位文章位置，比其他文本等更靠左一点。有6种大小（类型），第一种最大，第四种跟普通加粗文本差不多，第六种最小，可以当做标题等级。

    示例：


    只因块旋转一分钟

    只因块旋转十分钟

    只因块旋转一小时

    只因块旋转十小时

    只因块旋转一整天
    只因块旋转一个世纪

    代码：

    最大大小（一级标题）

    <h1 id="value">内容</h1>

    第二大大小（二级标题）

    <h2 id="value">内容</h2>

    其实第几级标题就是改一下元素名中h后面的数字。以此类推到h6。当然h4以下的比普通文本还小就不怎么常用了。

    提醒：h3及一下（包括h3）就不要用id属性了，下面就会讲到；如果用在提示框（页首提示框，尽管后来被用作其他用途）中，请加一条style属性，并将值设置为text-indent:0;（一定要注意）。

    属性id的使用：

    • 用于在目录中定位，设置了id值就能在目录中放置锚链接定位，后面会讲到。要使用的话直接设置为这个标题的文本内容即可。

  • 普通文本段落

    普通的文本段落，固定至少占一行。

    示例：


    哪个小朋友半夜不睡觉？哪个小朋友半夜不睡觉我就到你家里抓你！把你抓走！睡不睡！！！


    代码：

    <p class="value">内容</p>

    属性class可用的值：

    • caution 加粗显示。
    • warning 加粗+斜体+标红+下划线显示，多用于需要引起注意的重要事项。
    • explain 颜色减淡+斜体显示，多用于不怎么重要的解释性文本。
    • delete 颜色再减淡，并加以删除线，多带有娱乐性和调侃性。
    • hidden 直接隐藏，文本颜色与背景颜色相同，需要在电脑端上选中文本才能看见。直接说出来不是特别好或者隐藏起来更有趣的可以使用。
    • code 代码块，字体为代码字体，之所以叫代码块也是因为设计了边框，页首提示框和下载按钮的边框都是参照这个的边框的。

    • cbtext 应用于主页中的分类块中的值，由于词条中不需要用到分类块，且这个值本质上就是缩进32像素，使用在词条中相当不和谐，所以不要使用。

  • 行内文本段

    在普通的文本段落内使用，用于一行文本内个别文字使用不同样式。

    因为一个普通文本段落里所有文本的样式都是一样的（不是说每个字长得一样，是说使用的外观样式，你可以理解为字体或者特效，外观样式是一样的），要在一行里面为单独的一段文本使用不同的样式，就需要用到这个了。

    示例：（在普通文本段落中演示）


    大外孙你牛波一，你的消费可不低


    代码：

    <父元素>省略的父元素的文本内容<span class="value">内容</span>省略的父元素的文本内容</父元素>

    属性class可用的值：

    • caution 加粗显示。
    • warning 加粗+斜体+标红+下划线显示，多用于需要引起注意的重要事项。
    • explain 颜色减淡+斜体显示，多用于不怎么重要的解释性文本。
    • delete 颜色再减淡，并加以删除线，多带有娱乐性和调侃性。
    • hidden 直接隐藏，同普通文本段落中的hidden属性。
    • code 代码段，字体为代码字体，也有边框。

      • 用于代码块或者代码段中的代码中的特殊行内文本段样式：

      • codeint 标蓝色的内容，最初被设定为用来显示代码中的数字、色号等，后用于其他不同的用途。
      • codestring 标绿色且加粗的内容，一开始用来显示代码中的字符串，后用于其他不同的用途，但大多离不开字符串及特殊字符数据。
      • codefunc 标紫色的内容，本来是用来显示代码中的函数的，现在变成了次一级的标注。
      • codeidtf 标红色且加粗的内容，原本是代码中的保留字关键字，你也懂的后来发生了什么。
      • codeexplain 标灰色并使用斜体的内容，用来表示注释、注释性文本和省略的代码。
