上面标了蓝色的是要替换成你自己的东西：

注意保存的时候名称就用你的词条名后面加个.html，文件名下面的“保存类型”点一下，弹出来的列表里选择“所有类型（*.*）”。

注意事项

词条名的命名有一定限制。

  • 名称不能超过30个字符（包括所有类型的符号，以在页面显示的内容为准）
  • 名称如果包含连续的空格，则需要把空格（每一个空格）替换成转义字符实体&nbsp;。此外在名称中最好还是要替换的有：
    • # → &#35;
    • / → &#47;
    • . → &#46;
  • 不要使用像“这样做香翅捞饭，鸡肉咸香又带有一丝甘甜，太下饭了！”这样的像短视频和博文小文章的诡异标题。你想想百科词条该用什么名字。
  • 名称里不能包括换行。你现在也应该不会在HTML里换行才对。
  • 教程类词条标题格式应该是“教程：（你的教程名）”，就像这个教程一样。
  • 避开使用emoji表情、怪异或者少见不常用的符号、Tab、链接、有误导性或法律性的符号 、网址等。

编写内容

注：每行代码前面请添加20个空格保持对齐。代码中的空格也需要添加。

所有部件（元素）的代码（注释不是部件元素）其实除了部件内容外代码形式都是挺相似的。下面介绍一下部件代码都是如何组成的。

普通形式

<元素名 属性名="属性值">内容</元素名>

无内容纯标签形式

<元素名 />

嵌套形式

<元素名 属性名="属性值">
    内容（可以是元素标签代码）
</元素名>

上面的“属性名”和“属性值”是成对的，以下简称属性。

一般来说一个元素可以没有属性，不一定要写属性；一个元素也可以有多个属性，不同属性之间用空格隔开来；但是某些特殊的就一定得有属性，一定要有属性的下面会讲到。

属性有很多种，每一种的功能也当然是不一样的，下面介绍的元素也都会写可以用哪些属性或必须用哪些属性，以及使用的场合。

元素（部件）代码列表

标红色的是必要属性、紫色的是可选属性、蓝色的是内容。属性值里的“value”则是可以替换值的地方。

文本类

标题文本

用于标题的文本，亦可用于在目录中定位文章位置，比其他文本等更靠左一点。有6种大小（类型），第一种最大，第四种跟普通加粗文本差不多，第六种最小，可以当做标题等级。

示例：

只因块旋转一分钟

只因块旋转十分钟

只因块旋转一小时

只因块旋转十小时

只因块旋转一整天
只因块旋转一个世纪

代码：

最大大小（一级标题）

<h1 id="value">内容</h1>

第二大大小（二级标题）

<h2 id="value">内容</h2>

其实第几级标题就是改一下元素名中h后面的数字。以此类推到h6。当然h4以下的比普通文本还小就不怎么常用了。

提醒：h3及一下（包括h3）就不要用id属性了，下面就会讲到；如果用在提示框（页首提示框，尽管后来被用作其他用途）中，请加一条style属性，并将值设置为text-indent:0;（每个提示框里的标题文本都要加，一定要注意）。

属性id的使用：

  • 用于在目录中定位，设置了id值就能在目录中放置锚链接定位，后面会讲到。要使用的话直接设置为这个标题的文本内容即可。
普通文本段落

普通的文本段落，固定至少占一行。

示例：

哪个小朋友半夜不睡觉？哪个小朋友半夜不睡觉我就到你家里抓你！把你抓走！睡不睡！！！

代码：

<p class="value">内容</p>

属性class可用的值：

  • caution 加粗显示。
  • warning 加粗+斜体+标红+下划线显示，多用于需要引起注意的重要事项。
  • explain 颜色减淡+斜体显示，多用于不怎么重要的解释性文本。
  • delete 颜色再减淡，并加以删除线，多带有娱乐性和调侃性。
  • hidden 直接隐藏，文本颜色与背景颜色相同，需要在电脑端上选中文本才能看见。直接说出来不是特别好或者隐藏起来更有趣的可以使用。
  • code 代码块，字体为代码字体，之所以叫代码块也是因为设计了边框，页首提示框和下载按钮的边框都是参照这个的边框的。

  • cbtext 应用于主页中的分类块中的值，由于词条中不需要用到分类块，且这个值本质上就是缩进32像素，使用在词条中相当不和谐，所以不要使用。

行内文本段

在普通的文本段落内使用，用于一行文本内个别文字使用不同样式。

因为一个普通文本段落里所有文本的样式都是一样的（不是说每个字长得一样，是说使用的外观样式，你可以理解为字体或者特效，外观样式是一样的），要在一行里面为单独的一段文本使用不同的样式，就需要用到这个了。

示例：（在普通文本段落中演示）

大外孙你牛波一，你的消费可不低

代码：

<父元素>省略的父元素的文本内容<span class="value">内容</span>省略的父元素的文本内容</父元素>

属性class可用的值：

  • caution 加粗显示。
  • warning 加粗+斜体+标红+下划线显示，多用于需要引起注意的重要事项。
  • explain 颜色减淡+斜体显示，多用于不怎么重要的解释性文本。
  • delete 颜色再减淡，并加以删除线，多带有娱乐性和调侃性。
  • hidden 直接隐藏，同普通文本段落中的hidden属性。
  • code 代码段，字体为代码字体，也有边框。

    • 用于代码块或者代码段中的代码中的特殊行内文本段样式：

    • codeint 标蓝色的内容，最初被设定为用来显示代码中的数字、色号等，后用于其他不同的用途。
    • codestring 标绿色且加粗的内容，一开始用来显示代码中的字符串，后用于其他不同的用途，但大多离不开字符串及特殊字符数据。
    • codefunc 标紫色的内容，本来是用来显示代码中的函数的，现在变成了次一级的标注。
    • codeidtf 标红色且加粗的内容，原本是代码中的保留字关键字，你也懂的后来发生了什么。
    • codeexplain 标灰色并使用斜体的内容，用来表示注释、注释性文本和省略的代码。
超链接文本

用于超链接的文本，可以占一行也可以使用在一段文本中，点击链接后会传送到指定网址、文章指定位置，或者开始下载指定文件。

示例：

不用点我，我啥也不会干，只会在地址栏里面添加一个“#”。

代码：

<a href="value" class="value">内容</a>

属性href的使用：

有三种使用方法，分别是绝对路径（几乎不用）、相对路径（词条里很少用）和锚定位（绝大部分情况下仅在目录中使用，所以放在讲解目录的时候中讲解）

绝对路径：

直接设置为一个网址，用来访问非流汗黄豆百科站内的资源（访问站内资源用相对路径即可），现在几乎不需要用，目前没有任何一个页面有用到绝对路径。当然如果你的词条是借鉴外部网站上的某篇文章，你则需要带上原文链接。

示例：

GitHub官网 https://github.com

百度翻译 https://fanyi.baidu.com

CSDN官网 https://www.csdn.net

奇妙的一个邮箱网站 https://fuckyou.com

相对路径：

用于访问站内资源，词条中不怎么使用。要用一般也是用于该词条中的资源的获取。

一个词条所需要用到的资源（文件）比如图片，在词条里让大家下载的文件等等都需要整理在一个文件夹里的。访问这些文件就是用相对路径。

相对路径的路径是相对于文件自身的，下面所说的起始文件就是词条文件自身。

使用

  • 跟起始文件处在同一目录（文件夹）中的文件或者文件夹，直接输入该文件/文件夹的名字即可。如果是一个文件，请加上扩展名。
  • 如果像上面那样访问了一个同级目录下的文件夹，想要访问文件夹的内容，请在文件夹的名字后面加一个斜杠，并加上文件/文件夹的名字。
  • 想访问上级目录的内容（一般不会），输入../加上文件或者文件夹名即可。

示例

访问同一目录的一个叫做“词条.src”的文件夹（目录）内的另一个叫做“图片二”的jpg图片：词条.src/图片二.jpg

访问同一目录下的文件夹“文件夹一”里面的文件夹“文件夹二”里面的叫做“图片三”的bmp图片：文件夹一/文件夹二/图片三.bmp

属性class可用的值：

  • sblink 用于侧边栏顶端的几个链接，词条中不要使用。
  • menuanchor 用于目录中的锚定位，讲目录的时候会讲到。
  • downloadbutton 用于下载按钮。使用这个的时候文本的开始和结束请各添加一个“↓”。

注：使用下载按钮时，请确保链接所指向的文件不能被浏览器直接打开。能被浏览器直接打开的不会创建下载任务，而是直接打开。浏览器可以直接打开网页文件、网页程序文件、图片、视频等。

如果确实是要下载这些类型的文件的话，可以将文件压缩为一个压缩文件，比如ZIP、RAR等。这种方式也适用于要下载多个文件的。

表格的代码相对来说比较复杂一点，还是需要讲解一下：

代码中的一个tr标签对应着表格里的一行，以下简称行。

一行里面的td标签对应着表格中一行中的一列，以下简称列。注意，这里的列不是表格中的一整列，实际上是类似与表格里的一个小方格。

上面的两种表格使用起来区别比较大，主要是在列的使用上。

第一种表格

第一种表格是一般用于介绍词条所讲述的内容的部分基本信息。

第一种表格对列的使用比较特殊：

每一行固定是有四列的，当然除了最后一行，最后一行可能只有两列。

表格中也是一个信息名和一个信息内容组成一对，在代码中则是一个class属性为key的列和一个class属性为value的列组成一对，简称一个键值对。

一个键值对正好就是两个列，所以第一种表格每一行固定只能是有四列或两列。

参照百度百科、360百科等百科的词条的基本信息表格。

你可以看看，参照一下本站的部件展示里面所展示的内容。

你应该注意，基本信息表格只使用键值对，你不应该使用下面所说的nrml列，这可能会对排版产生未知的伤害。

第二种表格

第二种表格对列的使用由于比较放松，也有需要讲解的：

还是一样，tr简称行，td简称列。

一般情况下最好是每一行的列的数量保持一致。每一行的列的宽度是相等的，顶多差一个像素，那个是整除不了的。

如果每一行的列的数量不相等可能会发生一些很鬼畜的问题。列最多的一行的每一列的宽度似乎会应用于所有列，哪怕某一行列没有那么多。

当然你如果实在想这么做，你可以看看下面讲的colspan和rowspan。

上面的属性里没有把值设定为value的就是固定的值。value只是用来提醒这里的值是要自己填成别的的。

列（td）的代码中有两个可选值：colspan和rowspan。这两个属性非常不常用，用来指定这个列跨越多少个格子：

colspan指定横向总共跨越（一般是向左）多少格、rowspan指定竖向总共跨越（一般是向下）多少格。跨越了一个格就是总共占了两个格。

如果向左一格向下一格同时出现，左下角一格也会合并进去。以此类推。

会跟上下内容产生一定量的间隔（16像素）。

示例：

代码：

<hr />

对，也不需要任何属性，也是没有属性。

代码注释

不会对网页产生任何影响，只用于给代码添加注释，一般不使用。

注释的代码跟上面的那些代码结构是不一样的。

注释不跟网页内容有关，没有示例。

代码：

<!--注释内容-->
